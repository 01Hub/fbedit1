Options



 Resource Editor Options.

● Name Export Format.

    Assembly equ's: IDC_BUTTONADD equ 1001
    C defines: #define IDC_BUTTONADD 1001
    hla: IDC_BUTTONADD := 1001;
    PureBasic: #IDC_BUTTONADD = 1001
    PowerBasic: %IDC_BUTTONADD = 1001
    User Defined: %name and %id will be replaced by the actual values.

● Name Export Output.

    File: The names are written to file.
    Clip Board: The names are pasted to clipboard.
    Output: The names are sendt to output window.

● Default Export Filename.

    Name of file if File is selected.

● Auto export when saved.

    If checked the export will be done when the project is saved.

● Grid X-Size, Grid Y-Size

    Sets the size of the grid.

● Grid color

    Click on the color box to change.

● Show Grid

    The grid will be visible.

● Snap To Grid

    While designing, controls will move and size according to grid.

● Show Size/Pos

    A tooltip will show size or position.

● Grid as lines

    The grid will be shown as lines.

● Styles as hex

    Control styles will be saved as hex values.

● Custom control column.

    This field contains the filename of a RadASM compatible custom control or a quoted string.
    Format RadASM control: filename.dll
    Example1: RAEdit.dll
    RadASM compatible code edit control.
    Format quoted string: "[dll],class,name,[caption],tooltip,width,height,style,exstyle"
    Example2: "Riched32.dll,richedit,IDC_RED,Text,RichEdit,100,100,503110C4,00000200"
    Old rich edit control.
    Example3: ",static,IDC_LINE,,Horizontal line,100,2,50000110,00000000"
    Horizontal line.

● Style mask column.

    This field contains the style mask. There can be max 3 masks.
    Format: Mask1[,Mask2[,Mask3]]
    Example: WS_,SS_

Here you can add control styles that RadASM normally does not support.
The Value and Mask must be entered as hex values.
Normally the Mask is the same as the Value, but if the style is part of a group the group value should be entered.

Here you can add resource types that are not initially supported by RadASM.
You can also select an editor for existing resource types.
NOTE!
Some columns are read only for resource types that RadASM initially supports.

● Name column.

    The resource name.
    Example: MTTEXT

● Value column.

    Normally 0, if user defined a value can be set.
    Example: 0

● Files column.

    Open files of type.
    Example: Ini (*.ini)|*.ini|Text (*.txt)|*.txt|All (*.*)|*.*
    NOTE!
    If left blank the resource type uses embedded file data.

● Editor column.

    Here you can specify an editor for the file.
    Example: notepad.exe
