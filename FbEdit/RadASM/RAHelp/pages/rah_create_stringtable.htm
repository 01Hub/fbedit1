Creating and editing string tables

String tables are used to store text in your program that can be used by the LoadString API call. RadASM has an integrated string table editor that automates the process of adding strings to your program. The IDE will automatically generate a new RC file for the strings as well as make the necessary #include in the main RC. All of the string labels are declared in the special RC file and it is independent of the main RC. Since RadASM actually stores the strings in the project file, though the string table RC is transportable to another project as is, it cannot be edited by the other project or it will be lost. See the RAP file entry for tips on copying string tables to other projects. It should be noted that as the LoadString API function will only accept integer parameters for the string table entry you must fill in the ID field of the String table editor. The string table editor function is similar to that of the Resource Editor.
