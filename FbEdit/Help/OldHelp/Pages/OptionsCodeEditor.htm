Code Editor Options



 

● Color Theme.
Here you can select a color theme.

Creating a new theme:
1. Select a theme close to what you want.
2. Change the colors you want.
3. Select New Theme from the combobox.
4. Give the theme a proper name and push Add New button.

● Colors.
Here you can choose the color for most FbEdit components.
For Comments, Strings, Operators and Numbers & hex you can also choose bold and italics font.

● Syntax.
Here you can choose colors and font options for each word group.
You add a new word to a group by typing the word in the textbox below active list and then push Add button.
There is also a tool to simplify moving words from one group to another.
You can also specify if the group belongs to RC file or Inline asm.

● Misc.
- Tab size
  Number of space characters.
- Line numbers
  If checked line numbers will be shown.
- Expand tabs
  Tabs will be expanded to spaces while typing.
- Single instance
  Only one instance of FbEdit is possible.
- Block complete
  The end of a block will automaticaly be added while typing.
- Save on make
  All files will be saved automatically on build.
- Autoload project
  The last project will be loaded on startup.
- Close on locks
  Tab will be closed on exit, even if locked.
- Backups
  The number of backups created.
- Auto indent
  Indent will be copied from the line abowe.
- Smart indent
  Indents will follow the rules for the block edited.
- Auto brace
  The end brace will automatically be inserted.
- Brace matching
  Braces will visualy be matched.
- Highlight active line
  Current line will be highlighted.
- Highlight comments
  Comments will be highlighted.
- Smart maths
  Typing ++ will convert to +=

● Fonts.
Here you can choose font for code editor and line numbers.

● Code Complete.
- Active
  A code complete list will be shown.
- Include / Inclib
  A code complete list with .bi or .a files will be shown.
- Auto Size
  Code complete list will autosize.
- Code Tooltip
  A tooltip will be shown on subs and functions.

● Case Convert.
- None
  No case converting will take place.
- Mixed case
  Case converting will follow the api file.
- Lower Case
  Case converting will be lower case.
- Upper Case
  Case converting will be upper case.
